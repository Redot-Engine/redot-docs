ImageFrames
===========

**Inherits:** :ref:`Resource<class_Resource>` **<** :ref:`RefCounted<class_RefCounted>` **<** :ref:`Object<class_Object>`

A container for sequence of :ref:`Image<class_Image>`\ s.

.. rst-class:: classref-introduction-group

Description
-----------

A container of :ref:`Image<class_Image>`\ s used to load and arrange a sequence of frames. Each frame can specify a delay for animated images.

Can be used to load animated image formats externally.

Supported animated image formats are `GIF <https://www.w3.org/Graphics/GIF/spec-gif89a.txt>`__ (``.gif``), `APNG <https://wiki.mozilla.org/APNG_Specification>`__ (``.png`` and ``.apng``), `WepP <https://developers.google.com/speed/webp/docs/riff_container>`__ (``.webp``), and any format exposed via a GDExtension plugin.

An :ref:`ImageTexture<class_ImageTexture>` is not meant to be operated from within the editor interface directly, and is mostly useful for rendering images on screen dynamically via code. If you need to generate images procedurally from within the editor, consider saving and importing images as custom texture resources implementing a new :ref:`EditorImportPlugin<class_EditorImportPlugin>`.

.. _class_ImageFrames_method_load:

.. rst-class:: classref-method

:ref:`Error<enum_@GlobalScope_Error>` **load**\ (\ path\: :ref:`String<class_String>`\ ) :ref:`ðŸ”—<class_ImageFrames_method_load>`

Loads a sequence of image frames from file ``path``.

\ **Warning:** This method should only be used in the editor or in cases when you need to load external images at run-time, such as images located at the ``user://`` directory, and may not work in exported projects.

::

    var frames = ImageFrames.load_from_file("res://animated.gif")
    var animated_texture = AnimatedTexture.create_from_image_frames(frames)
    $Sprite2D.texture = animated_texture

This way, textures can be created at run-time by loading images both from within the editor and externally.

\ **Warning:** Prefer to load imported textures with :ref:`@GDScript.load()<class_@GDScript_method_load>` over loading them from within the filesystem dynamically with :ref:`load()<class_ImageFrames_method_load>`, as it may not work in exported projects:

::

    var animated_texture = load("res://animated.gif")
    $Sprite2D.texture = texture

This is because images have to be imported as an :ref:`AnimatedTexture<class_AnimatedTexture>` first to be loaded with :ref:`@GDScript.load()<class_@GDScript_method_load>`. If you'd still like to load an animated image file just like any other :ref:`Resource<class_Resource>`, import it as an **ImageFrames** resource instead, and then load it normally using the :ref:`@GDScript.load()<class_@GDScript_method_load>` method.

\ **Note:** The image frame can be create from an imported texture using the :ref:`AnimatedTexture.create_from_image_frames()<class_AnimatedTexture_method_create_from_image_frames>` method:

::

    var texture = load("res://animated.gif")
    var image: AnimatedTexture = AnimatedTexture.create_from_image_frames(texture)
